# Markdown Documentation Explorer (VSCode Extension)

## Product Requirements Document (PRD)

Version: 1.0
Author: Nagarjun
Status: Draft

---

# 1. Overview

Markdown Documentation Explorer is a VSCode extension that automatically discovers, organizes, renders, and searches all markdown documentation in a repository.

The extension provides a centralized documentation experience inside VSCode, enabling developers to explore and navigate documentation similar to a documentation website (MkDocs / Docusaurus) without leaving the IDE.

The extension is designed for modern repositories where documentation is scattered across multiple directories and often generated by AI tools.

---

# 2. Problem Statement

Modern repositories contain documentation spread across multiple folders such as:

* docs/
* architecture/
* design/
* runbooks/
* adr/
* prompts/
* .ai/
* .cursor/
* .bmad-methods/

Challenges developers face:

* Difficulty locating documentation
* No centralized navigation
* Markdown preview is file-based only
* No cross-document search
* No hierarchy visualization
* No support for MkDocs navigation
* AI-generated documentation scattered in hidden folders

VSCode currently provides only basic markdown preview without repository-level documentation management.

---

# 3. Goals

The extension should:

1. Automatically discover markdown documentation in a workspace
2. Provide a documentation explorer sidebar
3. Render rich markdown previews
4. Support diagrams and markdown plugins
5. Provide global documentation search
6. Support MkDocs navigation structure
7. Support AI-generated documentation folders
8. Provide documentation analytics and health checks

---

# 4. Non Goals

The extension will not:

* Replace static documentation sites
* Manage documentation publishing
* Replace markdown editing tools

---

Primary Users

* Software Developers

# 6. Core Features

## 6.1 Workspace Markdown Discovery

The extension scans the workspace for markdown files.

Supported formats:

* .md
* .markdown
* .mdx

Includes:
1. normal folders
2. hidden folders
3. AI agent folders
4. documentation folders

Examples: 
docs/
design/
architecture/
.ai/
.cursor/
.bmad-methods/
.llm/
ai-docs/
adr/
runbooks/

Capabilities:

* Auto indexing of documentation
* Includes hidden folders
* Supports nested directories
* Optional inclusion of gitignored folders
* Automatic refresh on file changes and creation
* Git ignored folders optional
* Large repo optimized scanning

## 6.6 MkDocs Support

If a repository contains mkdocs.yml, the extension parses the navigation structure.

Example mkdocs configuration:

nav:

* Home: index.md
* Architecture:

  * Overview: architecture.md
  * API: api.md

The explorer should mirror this hierarchy.

Features:

* respect nav ordering
* nested navigation
* render docs as MkDocs structure


-> Below the documentation explorer, create one more window for MKDocs explorer that shows all the mkdocs.yml based on name in it and display the documentation preview when selected along with navigation inside preview.
